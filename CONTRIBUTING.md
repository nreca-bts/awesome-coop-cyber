# Contributing to Awesome Co-op Cybersecurity & Threat Intelligence

Thank you for your interest in contributing! 🎉  
This project is a curated list of high-quality cybersecurity, threat intelligence, and 
ICS/OT resources. Community contributions help keep it relevant, accurate, and valuable.

## How to Contribute

There are a couple ways you can suggest improvements or additions:

### 1. Submit a Pull Request (Preferred)

1. Fork this repository
2. Create a new branch (`feature/add-new-resource`)
3. Add your changes to the README using the format detailed below (to match existing)
4. Submit a pull request (PR)

Please include:
* A clear description of your addition or change
* Why the resource is valuable
* Any relevant tags: short, consistent with existing ones, not redundant to section headings
* Entry formatting that follows the table format used in the README, see example below.

```markdown
| Name                                     | Description                              | Tags          |
| ---------------------------------------- | ---------------------------------------- | ------------- |
| [Awesome Tool Name](https://example.com) | Short, clear description of the resource | `Tag1` `Tag2` |
```

### 2. Open an Issue (Accepted)

* Open a GitHub issue
* Use a clear title (e.g., *Add ICS threat modeling resource*)
* Include:
  * Resource name
  * Link
  * Short description
  * Suggested category

## Contribution Guidelines

To maintain quality and consistency, please follow these rules:

### ✅ What to Include

* High-quality, reputable resources you have direct experience with
* Actively maintained tools, frameworks, or lists
* Educational materials, research, or reference content
* Resources relevant to:
  * Cybersecurity
  * Threat intelligence
  * ICS / OT security
  * Critical infrastructure / electric utilities
  * Security culture and workforce wellbeing

### ❌ What Not to Include

* Low-quality or outdated resources
* Marketing-heavy or promotional content
* Broken or inaccessible links
* Duplicate entries

## Review Process

* Maintainers will review PRs for formatting, relevance, and quality
* You may be asked to revise your submission
* Once approved, your contribution will be merged 🎉

## Code of Conduct

Be respectful and constructive in all interactions.  
We aim to build an inclusive and helpful community.

## Thank You

Your contributions help improve the cybersecurity community for everyone. 💚
